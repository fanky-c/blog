# MyBlog Technical Guide

## 1. Project Overview

This project is a personal blog based on `Hexo + aircloud theme`.

- Source branch: `dev/blog`
- Output branch: `release/blog`
- Default branch: `master`

Core flow:

1. Write content in `source/`
2. Generate static files to `public/`
3. Deploy generated files to GitHub Pages branch by `hexo-deployer-git`

## 2. Tech Stack

- Runtime: `Node.js >= 14`
- Static site generator: `hexo@7`
- Theme: local theme `themes/aircloud`
- Template engine: `ejs`
- Markdown renderer: `marked`
- Search index: `hexo-generator-search` (`search.json`)

## 3. Directory Structure

```txt
.
├── _config.yml                  # Hexo main config
├── package.json                 # scripts and dependencies
├── scaffolds/                   # post/page/draft templates
├── source/                      # site source files
│   ├── _posts/                  # blog posts
│   ├── about/index.md           # About page
│   ├── tags/index.md            # Tags page
│   └── img/                     # image assets
├── themes/aircloud/             # local theme source
│   ├── _config.yml              # theme config
│   ├── layout/                  # ejs templates
│   └── source/                  # css/js/static for theme
├── public/                      # generated static output (do not edit manually)
└── .deploy_git/                 # deploy temp repo generated by Hexo
```

## 4. Local Development

### 4.1 Install dependencies

```bash
npm install
```

### 4.2 Start local server

```bash
npm run dev
```

Default local URL: `http://localhost:5001`

### 4.3 Build output

```bash
npm run build
```

Current `build` script:

```bash
hexo clean && hexo generate
```

This avoids stale cache issues and guarantees fresh output.

## 5. Content Development

### 5.1 Create a post

```bash
npx hexo new "文章标题"
```

Generated file: `source/_posts/<title>.md`

### 5.2 Post front-matter example

```yaml
---
title: 示例文章
date: 2026-03-08 10:00:00
tags:
  - js
  - node
---
```

### 5.3 Create custom page

```bash
npx hexo new page "about"
```

Then set layout in file front-matter, for example:

```yaml
---
layout: "about"
title: "About"
---
```

## 6. Theme Development

Main files:

- Layout entry: `themes/aircloud/layout/layout.ejs`
- Home list: `themes/aircloud/layout/index.ejs`
- Post page: `themes/aircloud/layout/post.ejs`
- Global styles: `themes/aircloud/source/css/aircloud.css`
- Global scripts: `themes/aircloud/source/js/index.js`

If you change theme files, run `npm run build` and verify generated files in `public/`.

## 7. Key Configurations

### 7.1 Hexo main config (`_config.yml`)

- Site metadata: `title`, `description`, `keywords`, `author`
- Routing:
  - `url`
  - `root`
  - `permalink`
- Pagination: `per_page`
- Theme: `theme: aircloud`
- Deploy target:
  - `deploy.type: git`
  - `deploy.repo`
  - `deploy.branch: release/blog`

### 7.2 Search config

```yml
search:
  path: search.json
  field: post
```

Frontend search relies on generated `public/search.json`.

## 8. Build and Deploy

### 8.1 Build only

```bash
npm run build
```

### 8.2 Deploy

```bash
npm run deploy
```

Deploy command internally runs:

```bash
hexo g -d
```

It generates files and pushes deployment output to configured Git branch.

## 9. Common Issues

### 9.1 Generated HTML/static files are empty (0 bytes)

Symptom:

- `public/*.html` or `public/js/*` size is `0`
- Pages appear blank in browser

Fix:

1. Use modern compatible dependencies (`hexo@7` and matching plugins)
2. Ensure Node version is `>=14`
3. Always run clean build:

```bash
npm run build
```

4. Verify output file sizes:

```bash
wc -c public/index.html public/js/index.js public/css/aircloud.css public/search.json
```

### 9.2 Search panel shows no result

Check:

1. `hexo-generator-search` is installed
2. `search` config exists in `_config.yml`
3. `public/search.json` exists and is not empty

## 10. Collaboration Conventions

- Do not edit files in `public/` manually
- Keep post assets under `source/img/`
- Keep article front-matter complete (`title/date/tags`)
- Verify locally before deploy:
  - `npm run build`
  - `npm run dev`

## 11. Suggested Release Checklist

1. Install deps: `npm install`
2. Validate config changes (`_config.yml`, theme config)
3. Build: `npm run build`
4. Smoke test home page, post page, tags, about, search
5. Deploy: `npm run deploy`
